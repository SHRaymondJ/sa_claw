# Product Repository Rules

## Repository Identity

当前目录是服装品牌导购工作台产品仓库，不再承担教学角色。

## Working Priorities

1. 先更新产品文档和任务板，再做实现
2. 所有后链路都必须使用真实实体 ID 和真实动作接口
3. 受限问答必须先经过导购域意图网关
4. 页面语言必须保持为正常业务产品语言
5. 敏感信息与真实业务数据不得进入仓库

## Delivery Standards

- 默认采用 SDD + TDD + E2E
- 前端协议层与实现层解耦
- 数据必须通过数据库和服务层访问
- 未注册协议统一降级，不得自由执行
- 页面更新、组件更新、协议更新必须同步更新文档

## Design Constraint

- 移动端优先，兼容 PC
- 服装零售风格
- 非圆角、细边框、克制层级、大留白
- 主界面是消息流工作台，不是传统后台表格页

## External Blockers

- Pencil 桌面连接当前失败，阻塞正式 `.pen` 设计文件输出
- 在阻塞解除前，允许先推进 PRD、技术设计、任务板和代码实现，但必须在文档中记录该阻塞
