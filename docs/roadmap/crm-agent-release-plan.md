# 商业化导购 Agent 版本发布计划

## 分支模型

- `main`：稳定发布分支
- `develop`：开发集成分支
- `test`：验收与回归分支
- `codex/v{版本号}-{任务短名}`：功能任务分支

## 版本切分

### `v0.1.0` 工程基线

- 文档体系落地
- 分支策略固定
- 发布门禁固定
- 远端仓库初始化
- 当前系统基线测试和首轮推送

### `v1.0.0` 去写死与工程底座

- 结果稳定性契约
- 决策分层
- 关键硬编码迁出
- 测试断言重构

### `v1.1.0` 语义测试与评测

- 重复提问测试
- DB 对照测试
- 生成变化语义评测
- 失败样本沉淀

### `v1.2.0` 受控生成式 UI

- 复合语义组件
- schema validator

- 协议版本化

### `v1.3.0` Skills 与学习治理

- 产品内技能层

- 回滚与灰度

## 发版门禁

每个版本都必须通过：

- `pytest`
- `pnpm test -- --run`
- `pnpm build`
- `pnpm exec playwright test`
- `test` 环境回归清单
- `main` 环境上线后 smoke test

## 版本发布流程

1. 从 `develop` 切功能分支
2. 小任务 TDD 完成并验证
3. 合回 `develop`
4. 大版本完成后从 `develop` 合到 `test`
5. `test` 环境完整回归
6. 从 `test` 合到 `main`
7. `main` 环境上线后回归
8. 打 tag，写发布记录和版本复盘

## 回滚原则

- `test` 环境发现阻断问题，不进 `main`
- `main` 环境发现 P0 或 P1，优先回滚到上一版本 tag
- 所有回滚都要补复盘和回归用例
