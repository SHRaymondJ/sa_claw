# 服装品牌导购工作台 PRD

## 1. 产品定位

面向门店导购的移动端优先工作台。导购通过自然语言输入目标，系统返回可执行的客户建议、商品建议、跟进任务和私聊话术，并提供可点击的详情与动作闭环。

## 2. 一期目标

- 让导购在一个页面内完成“提问、筛选、查看、执行、回写”
- 输出可解释、可追踪、可执行的结果
- 保证返回内容来自数据库与已实现接口能力
- 在业务展示层隐藏底层技术词汇

## 3. 非目标

- 不做完整登录体系
- 不做多品牌通用平台
- 不做互联网搜索
- 不做向量检索
- 不做真实外部消息发送

## 4. 目标用户

- 门店导购
- 店长或区域运营演示人员
- 内部产品、研发和业务评审人员

## 5. 核心场景

### 场景 A：找今日优先跟进客户

输入示例：

`帮我找今天该跟进但还没联系的高净值客户`

返回：

- 客户卡片列表
- 推荐理由
- 下一步建议
- 话术卡片
- 操作轨迹

### 场景 B：按客户偏好推荐商品

输入示例：

`给最近关注通勤西装的客户挑 3 款本周有货的单品`

返回：

- 商品卡片
- 库存状态
- 适配客户标签
- 推荐说明

### 场景 C：处理未完成任务

输入示例：

`把今天到期还没完成的回访任务按优先级排一下`

返回：

- 任务卡片
- 优先级说明
- 可直接完成或查看详情的动作

## 6. 页面结构

### 主页面

- 顶部状态条：导购身份、门店、今日待办、当前会话状态
- 消息流：用户输入、系统回复、结果卡片
- 输入区：文本输入、快捷问题、发送按钮
- 详情层：移动端抽屉，PC 端持久右栏

### 解释页

- 协议层
- 组件映射
- 数据流
- 安全边界
- 后续更新机制

## 7. 关键能力

- 协议化 UI 渲染
- 动作分发与实体详情
- 导购域意图网关
- 受限回答
- 数据驱动推荐
- 会话持久化
- 任务状态更新
- 拒答卡片

## 8. 数据范围

- 客户：200-500
- 商品：80-150
- 跟进任务：30-50
- 带客户标签、偏好、互动记录、库存状态、商品图和来源字段

## 9. 成功标准

- 导购域问题可以稳定返回卡片与动作
- 点击客户、商品、任务卡都能打开真实详情
- 任务状态更新后消息流和详情同步
- 非导购域问题统一拒答
- 页面在移动端与 PC 上都可用

## 10. 风险

- Pencil 设计连接阻塞，正式设计稿产出可能滞后
- 若真实模型不可用，需要保证 mock 降级仍能演示完整闭环
- 图片素材若改为本地托管，需额外处理体积与来源追踪
