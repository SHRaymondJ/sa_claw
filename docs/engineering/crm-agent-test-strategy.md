# CRM Agent 测试策略

## 测试目标

把测试从“固定答案校验”升级成“业务事实稳定 + 生成语义可信”的 agent 产品测试。

## 测试分类

### 1. 事实稳定测试

适用问题：

- 客户筛选
- 品类盘点
- 库存查询
- 明确约束过滤

要求：

- 数据库不变时，实体集合稳定
- 默认排序稳定
- 摘要文本可以轻微变化，但不能改变结论

### 2. 生成变化测试

适用问题：

- 商品推荐理由
- 执行摘要
- 沟通话术

要求：

- 核心实体结果稳定
- 理由文本允许变化
- 不允许编造数据库外信息

### 3. 重复提问策略测试

要求覆盖：

- 同会话重复提问默认保持主结果
- 出现“不满意/换一批/别的”信号后进入换批
- 动作后或数据变化后再次查询应刷新

### 4. DB 对照测试

同一句问题在数据库变更前后分别执行：

- 验证结果变化来自数据事实，而不是随机漂移

### 5. 生成式 UI 测试

- schema 合法
- 复合组件渲染正确
- 未知 schema 可安全降级
- 后链路动作真实可用

### 6. 商业治理测试

- 权限
- 多租户隔离
- 审计
- 限流
- 灰度和降级开关

## 当前必须覆盖的示例

- `帮我找一下最近看过西装但是没有买的客户`
- `给乔知夏推荐几款商品，并且给出推荐理由`
- `我要维护一下乔安禾的客户关系`
- `按她的喜好继续推荐`
- `换一批`
- `说说优衣库和政治新闻`

## 执行纪律

- 一个小任务做完就跑对应测试
- 大版本进入 `test` 前必须跑完整套
- `main` 合并后必须做线上 smoke test
