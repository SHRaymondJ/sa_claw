# CRM Agent TDD 执行标准

## 原则

- 一个小任务只解决一个问题
- 先写测试，再做实现；至少先补失败用例
- 每个小任务完成后立刻验证，不攒着一起测
- agent 产品允许生成表达变化，但业务事实必须严谨可测

## 测试分层

### 1. 后端单测

- 规则判断
- 上下文解析
- 状态迁移
- 稳定性契约
- schema 选择

### 2. 后端集成测试

- API 返回结构
- 数据库状态变化
- 重复提问策略
- 动作后查询变化

### 3. 前端组件测试

- renderer 渲染
- action registry
- fallback 渲染
- 错误态和加载态

### 4. E2E

- 聊天主链路
- 后链路详情
- 任务动作
- 多轮追问
- 切客户
- 切任务
- 澄清态

### 5. 语义评测

- 生成理由是否围绕真实实体
- 生成话术是否引用真实客户和商品事实
- 是否越界或幻觉

## 通过标准

- 稳定型问题：实体结果必须稳定
- 生成型问题：实体结果稳定，文本语义通过
- 失败样本必须补进回归

## 禁止事项

- 禁止以“生成内容不能严格比较”为理由跳过测试
- 禁止只跑单测不跑 E2E 就宣称完成
- 禁止在测试失败时直接修改断言去迎合错误实现
