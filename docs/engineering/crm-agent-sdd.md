# CRM Agent SDD 执行规范

## 目标

所有重要改动都先有规格，再有实现，避免继续在聊天主链路上边猜边改。

## 标准流程

1. 明确版本目标
2. 更新技术设计和任务板
3. 确认本次改动影响：
   - 业务策略
   - UI 协议
   - 会话状态
   - 测试口径
4. 写清楚验收标准
5. 再开始 TDD 实现

## 必须先写清楚的内容

- 当前问题是什么
- 这次改动属于：
  - Bug 修复
  - 稳定化
  - 体验优化
  - 协议演进
  - 治理能力
- 影响哪些接口、组件、状态和测试
- 是否会影响历史会话兼容
- 哪些结果必须稳定，哪些允许生成变化

## 变更控制

- 需求变化先改文档，不允许直接改代码
- 新增协议字段必须写到 UI schema 文档
- 新增状态字段必须写到技术设计
- 新增测试口径必须写到测试策略

## 禁止事项

- 禁止把业务结论写死在 prompt、常量、切片和默认文案里
- 禁止在没有测试口径的情况下修改聊天主链路
- 禁止用“先跑通再补文档”的方式改核心流程
