## 当前已记录问题

### Demo 时期大量写死策略

- 发现版本：`v0.1.0` 前
- 场景：客户筛选、推荐数量、固定组件组合
- 现象：系统行为更像固定模板，而不是商业化导购产品
- 复现步骤：连续提问筛选和推荐类问题时，可看到固定数量、固定组合、固定表达倾向
- 根因分类：计划问题 + 执行问题
- 修复动作：纳入 `v1.0.0` 去写死与工程底座
- 是否补进回归：是
- 当前状态：待修复

### 重复提问被缓存误判为 fresh

- 发现版本：`v1.0.0` 开发中
- 场景：同会话重复问同一个推荐问题
- 现象：系统直接命中上一轮缓存，导致 `repeat_query_mode` 一直显示为 `fresh`
- 复现步骤：连续两次提问“给乔知夏推荐一些产品，并且说明理由”
- 根因分类：执行问题
- 修复动作：重复提问模式下绕过前置缓存，并新增 `preserve/diversify` 测试
- 是否补进回归：是
- 当前状态：已修复

### 客户总览样本数和列表结果不一致

- 发现版本：`v1.0.0` 开发中
- 场景：配置 `CRM_CUSTOMER_SAMPLE_LIMIT`
- 现象：概览卡的 `sample_limit` 已更新，但客户列表仍按旧默认值截断
- 复现步骤：设置 `CRM_CUSTOMER_SAMPLE_LIMIT=6` 后请求“现在有哪些客户”
- 根因分类：执行问题
- 修复动作：客户总览查询改为显式使用配置样本数，并补配置回归测试
- 是否补进回归：是
- 当前状态：已修复

### 可见结果和会话状态存在隐藏 `4` 条截断

- 发现版本：`v1.0.0` 开发中
- 场景：商品推荐、客户维护、任务结果回写会话状态
- 现象：前端已展示完整结果，但 `focus_scope`、`last_entity_ids` 和部分测试仍按旧的 `4` 条截断
- 复现步骤：提问“找6件适合夏天穿的衣服”或进入客户维护链路，检查响应里的 `product_grid` 与 `session_snapshot`
- 根因分类：执行问题 + 测试缺失
- 修复动作：统一按当前可见商品/任务集合写入响应组件、`focus_scope`、`session_snapshot` 和最终实体集合，并补后端回归测试与前端/E2E 断言修正
- 是否补进回归：是
- 当前状态：已修复

### 后端协议新增时，前端 registry 可能漏注册

- 发现版本：`v1.0.0` 开发中
- 场景：后端新增 `component_type` 或 `action_type`
- 现象：如果前端没有同步注册，页面只能降级成未注册组件，或者动作直接无响应

- 根因分类：测试缺失
- 修复动作：新增协议同步测试，直接从后端源码提取当前协议白名单，并校验前端 renderer/action registry 已全部覆盖
